## 四、精度与正确性对比

### 4.1 精度保证矩阵

| 方法 | 存在即返回真 | 不存在一定返回假 | 描述 |
|------|-------------|----------------|------|
| Bitmap | ✅ 是 | ✅ 是 | 数学保证 100% 正确 |
| Bloom | ❌ 可能误判 | ✅ 是 | "不存在的一定不存在，存在的可能不存在" |
| SQLite | ✅ 是 | ✅ 是 | ACID 保证正确 |
| Tree | ✅ 是 | ✅ 是 | 结构保证正确 |
| MixTree | ✅ 是 | ✅ 是 | 分层结构保证正确 |

### 4.2 布隆过滤器假阳性数学分析

误判率公式：

```
P ≈ (1 - e^(-k * n / m))^k

其中：
- n: 插入元素数量
- m: 位图大小(bits)
- k: 哈希函数数量
```

本项目配置：
- n = 10亿
- m = 9.58亿 bits
- k = 7
- **理论误判率 ≈ 1%**

## 五、技术特性深度对比

### 5.1 数据结构本质

```
Bitmap:     数学压缩的整数集合表示
            ↓ 每个号码对应一个bit位

Bloom:      概率型数据结构，k个哈希函数映射
            ↓ 牺牲精度换空间

SQLite:     B+树索引 + 磁盘持久化
            ↓ 牺牲性能换空间

Tree:       前缀共享的Trie树结构
            ↓ 牺牲内存换纯粹设计

MixTree:    前7层基数树 + 后4层位图
            ↓ 分层优化，性能 × 空间 的最优平衡点
```

### 5.2 并发与扩展性

| 特性 | Bitmap | Bloom | SQLite | Tree | MixTree |
|------|--------|-------|--------|------|---------|
| 只读并发安全 | ✅ | ✅ | ✅ 连接池 | ✅ | ✅ |
| 写入并发安全 | ❌ | ❌ | ✅ 事务 | ❌ | ❌ |
| 横向扩展能力 | ⚠️ 分片 | ⚠️ 分片 | ✅ 主从 | ❌ | ❌ |
| 冷启动速度 | 慢 | 中 | 极快 | 极慢 | 极慢 |

## 七、优缺点总结

### 7.1 Bitmap 位图

✅ **优点：**
- 数学保证精确
- 查询速度最快 (~10ns)
- 序列化支持最好
- 内存占用可控 (400MB)

❌ **缺点：**
- 稀疏数据集压缩率下降
- 不支持前缀统计
- 启动加载需2-3秒

---

### 7.2 Bloom 布隆过滤器

✅ **优点：**
- 内存占用最小 (~115MB)
- 查询速度极快 (~50ns)
- 实现最简单

❌ **缺点：**
- 1% 假阳性率
- 无法删除元素
- 无法列举所有元素

---

### 7.3 SQLite 数据库

✅ **优点：**
- 内存占用可忽略 (<10MB)
- 启动最快 (<100ms)
- 标准SQL接口
- 支持增量更新

❌ **缺点：**
- 查询最慢 (毫秒级)
- 需要CGO支持
- 磁盘IO瓶颈

---

### 7.4 Tree 基数树

✅ **优点：**
- 设计纯粹优雅
- 支持前缀统计
- 天然支持号段分析
- 查询与数据量无关

❌ **缺点：**
- 内存占用巨大 (20GB+)
- 无序列化支持
- 启动极慢

---

### 7.5 MixTree 混合树 🏆 综合最优

✅ **优点：**
- 空间优化极致 (相比Tree省50%+)
- 实际号段仅需6.5MB！
- 查询速度极快 (~40ns)
- 100%精确无假阳性
- 支持前缀统计功能
- 综合性能最佳

❌ **缺点：**
- 实现复杂度较高
- 无序列化支持
- 极端分散号段优势减弱

---

## 九、最终结论

### 🏆 综合冠军：MixTree 混合树

**在绝大多数业务场景下，MixTree 都是最优选择：**

1. **空间效率**：相比纯基数树节省 50%+ 内存
2. **查询性能**：~40ns，单线程每秒 2500 万次查询
3. **正确性**：100% 精确，无任何假阳性/假阴性
4. **附加能力**：内置号段前缀统计功能
5. **实际奇迹**：真实运营商号段下仅需 **6.5MB** 内存！

### 边界情况：

- **极端内存限制 + 可接受假阳性** → Bloom
- **极端内存限制 + 必须精确 + 查询极低频** → SQLite
- **容器内存限制 2GB + 高并发** → Bitmap
- **纯粹学术研究** → 纯基数树
